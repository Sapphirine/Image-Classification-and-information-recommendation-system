Problem 1 (5 points)


Consider a database of two items A and B. Initial values are A = B = 0 and the consistency requirement is A = 0 or B = 0. Transactions T1 and T2 are coded as follows, where a and b are local variables:

T1: {a=R(A); b=R(B); if (a==0) {W(B, b+1);}}
T2: {b=R(B); a=R(A); if (b==0) {W(A, a+1);}}

  1. Show that these transactions are correct by showing that every serial execution involving these two transactions preserves the consistency of the database.
  2. Show a concurrent execution of T1 and T2 which produces a non serializable execution.
  3. Is it possible that T1 and T2 would run concurrently under Strict 2PL? If yes, show such a schedule; if no, explain why not.   


Problem 2 (5 points)

Assume the schedule

S: W4(y) R1(x) R1(w) W1(w) W2(x) C2 W3(w) C3 W4(w) C4 W1(z) C1
Where Rn(x)/Wn(x) indicates transaction Tn reads/writes data item x, and Cn indicates Tn commits.

  1. Show the precedence graph of S.
  2. Prove S is correct, i.e., serializable.
  3. Enumerate all serial schedules that are equivalent to S.
